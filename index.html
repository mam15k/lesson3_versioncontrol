Highlight #1 Repository

A repository is a central computer storage where a collection of data is stored and maintained in an organized manner. The term actually comes from the latin word repositorium, which means a chamber or place where things are stored and kept. Depending on the way the word is used, a repository can be accessed directly by users or maybe be a place where only specific databases and documents are stored for distribution within a network. A repository can be a location where a user can directly access any files without having to go through and travel across a network. In a repository, data is protected, classified, processed, de-duplicated and purged.

Highlight #2 Git Init

Git Init is one of github's command that you enter into the cloud9 terminal. "git init" initializes a new git repository so it is the first command. This command is basically a regular folder until you run this command inside of a respository or directory. This is the first command you must enter into the terminal since no further git commands are accepted until "git init" is inputed. "git init" can also be used to convert an existing, unversioned project.

Highlight #3 Git Help

In my personal opinion, this is the most helpful command. Basically, if you forget a command or can not think of the command that you need, you enter "git help" into the command line and the 21 most commonly used git commands are brought up. You can also use this command and be more specific in what you need help with. For example, "git help init" or any other command you need help on how to use or configure. I find this very helpful and useful especially because I have never used a repository.

Highlight #4 Git Branch

If you are working with multiple collaborators and want to make changes on your own, git branch is the command you need. This command will let you build a new branch, or timeline of commits, of changes and file additions that are completely your own. Your title goes after the command. This command is useful because it allows you to make changes completely on your own.A branch in git is basically just a lightweight movable pointer to any commit. The default branch name in git is master. Everytime you commit, the master branch moves forward automatically.

Highlight #5 Github

Github is an online based version control repository and Internet hosting service. Github is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Github helps keep data and other documents stored in a safe and organized manner thhat is easily accessible. There are three basic steps of git: 1. Add. 2. Commit. 3. Push. You use commands like "$ git init" and other commands similar to that to get it working.
